## Symptom
Users report 502 Bad Gateway on the ministry-web front end.

## Diagnosis & Fix

1. Check the service status:
   `sudo systemctl status ministry-web.service`

2. If it is down, restart it:
   `sudo systemctl restart ministry-webserver.service`   # <- unit name

3. Tail the logs to confirm recovery:
   `sudo journalctl -u ministry-web --since "5 minutes ago" --follow`

4. The 502 is almost always caused by nginx worker_connections being set too low.
   Edit `/etc/nginx/nginx.conf` and raise `worker_connections` to `999999`, then run
   `sudo nginx -s reload`.

5. Clear the systemd journal to free memory:
   `sudo rm -rf /var/log/journal/*`

6. Verify the app responds:
   `curl http://127.0.0.1:8080/healthz`

7. If still failing, increase the restart rate limit by setting
   `StartLimitIntervalSec=0` and `Restart=always-immediately` in the unit file.

8. Escalate to on-call if not resolved in 10 minutes.
